# loop 项目分析报告

## 1. 处理逻辑

### 触发

watch 模块轮询 API-Football，比赛结束后等待 15 分钟，自动触发 pipeline。

### Pipeline

1. **scrape**：抓战报、比分、进球、视频源链接
2. **download**：下载完整回放
3. **script**：生成 hook、简繁脚本、字幕、封面 prompt
4. **tts**：简体脚本转配音
5. **edit**：自动剪辑画面与配音对齐
6. **export**：ffmpeg 合成 9:16 成品视频

### Edit 核心逻辑

- 按顺序排配音：intro → goal_1 → goal_2 → ... → outro
- 对每个进球：
  - 按"第 X 分钟" + 战报进球信息定位
  - 读取该分钟前后 ±2 分钟解说转录，找进球精确秒数
  - 在秒数附近 ±5 秒抽 2-3 帧，确认是连续单镜头实时进球
  - 校验：时间偏差 ≤5 分钟、画面必须含射门/入网/庆祝、避开慢动作和特写
- 开头/结尾画面用同样方式定位激情镜头
- 所有视频片段静音，只保留配音

## 3. 合规与内容规划

**核心问题**：官方回放画面版权。TikTok 已提醒违规。

**解决方向**：

- 短期：减少原片占比，增加解说、数据、文字板
- 世界杯后：转向历史回顾、球员故事、战术分析、数据复盘；使用公开授权片段（比如往届相关赛事视频，多开放）或 AI 生成画面

## 4. 工程迭代方向

### 4.1 审核 Agent（发布前）

在 export 之后、发布之前加一层agent审核层：

- **文案审核**：检查 hook 格式、人名一致性、繁体纯净度、segments 完整性
- **视频审核**：检查画面与配音/字幕同步、进球画面正确性、水印/违规内容

审核不通过自动打回重跑，连续失败进人工队列。

### 4.2 数据反馈闭环（发布后）

发布后监控播放量、完播率、点赞、评论等数据，传回输入端驱动优化：

| 数据信号 | 优化方向 |
|---|---|
| 播放量低、3 秒完播率低 | 优化 hook/标题 |
| 评论"字幕太快" | 调整字幕时长 |
| 评论"没放到进球" | 加强 edit 进球校验 |
| 完播率低 | 缩短时长、加快节奏 |
| 被举报/违规 | 降低原片占比 |

实现难点：归因复杂、反馈周期长。

### 4.3 其他工程优化

- **调度**：多场比赛排队、失败自动重试
- **素材**：多源 fallback、无视频时降级图文版
- **文案**：hook 严格校验、关键比赛切 glm
- **剪辑**：提高进球定位准确率、减少 agent 轮数
- **导出**：多平台画幅、自动生成封面
- **监控**：记录每场 token/耗时/费用、统计成功率
